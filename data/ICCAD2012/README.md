# ICCAD 2012 Benchmark Data

## Raw GDS Files

The original GDSII layout files are located in `raw/`, organized by chip index:

```
raw/
├── 1/
│   ├── train.gds
│   └── test.gds
├── 2/
│   ├── train.gds
│   └── test.gds
├── 3/
│   ├── train.gds
│   └── test.gds
├── 4/
│   ├── train.gds
│   └── test.gds
└── 5/
    ├── train.gds
    └── test.gds
```

Each GDS file contains three layers:
- **Layer 10/0** — Metal1 polygon shapes
- **Layer 21/0** — Hotspot (HS) marker bounding boxes (1200×1200 DBU)
- **Layer 23/0** — Non-hotspot (NHS) marker bounding boxes (1200×1200 DBU)

## Data Splits

Pre-defined train / val / test splits are in `splits/`:

| Split | Description |
|:------|:------------|
| `default/` | Standard train / val / test partition |
| `mini_balance/` | Balanced mini subset for quick experiments |
| `mini_imbalance/` | Imbalanced mini subset for stress testing |

Each split directory contains `train.csv`, `val.csv`, `test.csv` with columns:

```csv
filename,label
HS_00001,1
NHS_00001,0
```

## Derived Data

The following directories are generated by the preprocessing scripts and are **not** tracked in git:

| Directory | Script | Description |
|:----------|:-------|:------------|
| `points_4d/` | `scripts/preprocess/gds2point.py` | 4D point clouds (x, y, nx, ny) |
| `graphs/` | `scripts/preprocess/gds2graph.py` | PyTorch Geometric graph objects |
